## 一、業主的核心感受需求

目標體感：**訪客不是在「瀏覽」一份履歷，而是在「進入」一個真實的航管指揮中心。**

## 三、emons.de 帶來的體驗靈感

**網站**：https://emons.de/en  
**業主喜歡的部分**：帶入式遊戲感、沉浸氛圍感

### 值得借鑑的互動機制

| 機制 | emons.de 的做法 | V3 的應用方向 |
|------|----------------|--------------|
| **Scroll-driven 全屏切換** | 滾動控制服務區段過場，scroll-snap，像在推進關卡 | 進入 AI 機庫、企業核心等子頁面時，用 scroll 觸發 Three.js 場景轉換 |
| **WebGL + 動畫同步** | 3D 動畫與影片逐幀連動，沉浸感極強 | Three.js 地球旋轉與 UI 狀態同步（點擊產品 → 地球鏡頭拉近對應區域） |
| **浮動卡片解鎖** | 點擊才展開更多資訊，像「解鎖任務」 | AI 產品節點點擊 → 地球聚焦 + 右欄滑入架構解析，像解密任務書 |
| **空間感導航** | 使用者感覺在「穿越」不同空間，而非翻頁 | 各區塊切換用 Three.js 鏡頭移動而非頁面跳轉 |

## 四、V3 的體驗設計關鍵詞

- **沉浸式**（Immersive）：進入網站即進入航管中心世界
- **遊戲感**（Game-like）：解鎖、進場、任務、指揮——操作隱喻
- **電影級**（Cinematic）：Three.js 鏡頭運動、scroll-driven 場景切換
- **活的系統**（Living System）：雷達持續掃描、航班持續進場、狀態持續更新

---

## 五、Three.js 關鍵互動規格（業主期望）

1. **3D 地球可互動**：滑鼠拖曳旋轉、scroll 觸發鏡頭縮放
2. **飛行路徑動態**：AI 產品以光點沿弧線真實移動（Three.js QuadraticBezierCurve3）
3. **點擊連動**：點擊左欄產品 → Three.js 地球鏡頭移動至對應位置 → 右欄解鎖資訊
4. **Scroll 場景切換**：scroll-driven，不同區塊對應不同 Three.js 場景狀態
5. **Mobile 降級**：Three.js 替換為 CSS/SVG 靜態版，保留視覺氛圍
