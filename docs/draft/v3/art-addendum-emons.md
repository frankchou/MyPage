## A. Scroll 場景切換的氛圍過渡規格

### A.1 五個場景的光感定義

每個場景有其獨立的「環境光情緒」，由三個維度定義：Fresnel 顏色與強度、粒子場狀態、背景環境光強弱。

---

#### SCENE_IDLE（0%–15%，Command Center 首頁主場景）

**氛圍語意**：深夜指揮台，系統正在全速運行，一切都在掌控中。

| 維度 | 規格 |
|------|------|
| Fresnel shader 顏色 | `vec3(0.0, 0.784, 0.831)` — 對應 `--color-cyan` `#00C8D4` 的 float 值 |
| Fresnel intensity | 1.4（art-direction.md 5.3 節基準值，此場景為正常基準） |
| Fresnel 範圍（power 值） | `pow(1.0 - dot(vNormal, vViewDir), 3.0)`，power 3.0，邊緣清晰 |
| 粒子場密度 | 100%（基準，100–200 個環境漂移粒子，art-direction.md 6.2 節規格） |
| 粒子色調 | 90% 近白 `#E8EDF2`（透明 0.6–0.9），5% 琥珀 `#D4940A`（透明 0.4），5% 霓虹青 `#00C8D4`（透明 0.3） |
| 粒子漂移速度 | 極緩，0.008 場景單位 / 幀 |
| 環境光（AmbientLight） | color `#1A2A4A`，intensity 0.30（基準，維持夜間氛圍） |
| 地面光暈 | 底部 15% 漸層，`rgba(212,148,10,0.06)` 至透明（跑道燈隱喻，基準值） |
| 整體暗房強度 | 中等（背景光源 intensity 總和 ≤ 1.4） |

---

#### SCENE_APPROACH（16%–45%，進入機庫過場）

**氛圍語意**：飛機開始下降，鏡頭推近，高度感降低，空氣變得密實。場景是「正在進入某個具體的地方」。

| 維度 | 規格 |
|------|------|
| Fresnel shader 顏色 | 從 `#00C8D4`（cyan）漸變至 `#008A93`（cyan-dim），反映鏡頭推近、距離感減少 |
| Fresnel intensity | 從 1.4 線性過渡至 1.0（鏡頭推近，大氣層邊緣在視角中「後退」） |
| Fresnel power | 從 3.0 漸增至 3.8（邊緣更銳，強調接近感） |
| 粒子場密度 | 降至 60%（過場感，粒子稀薄暗示「正在穿越」） |
| 粒子漂移速度 | 從 0.008 加速至 0.014（空氣流動感） |
| 粒子色調偏移 | 琥珀比例從 5% 升至 12%（準備進入下一場景的預熱） |
| 環境光 | intensity 從 0.30 降至 0.22（暗化，靠近的密閉感） |
| 地面光暈 | 強度從 `rgba(212,148,10,0.06)` 升至 `rgba(212,148,10,0.10)`（光暈微亮，暗示降落目標在下方） |
| 整體暗房強度 | 略深（背景收暗，焦點移向地球表面） |

---

#### SCENE_SKUNKWORKS（46%–70%，AI 機庫）

**氛圍語意**：秘密研發基地，琥珀金的工業鍛造感，地球快速旋轉暗示能量密度高。emons.de 的「關卡解鎖」感在此最濃。

| 維度 | 規格 |
|------|------|
| Fresnel shader 顏色 | 從 `#008A93`（cyan-dim）過渡至 `vec3(0.831, 0.576, 0.039)` — 對應 `--color-amber` `#D4940A` 的 float 值 |
| Fresnel intensity | 1.0 升至 1.6（場景最高 Fresnel，琥珀色邊緣光，工業感強） |
| Fresnel power | 3.8 降至 2.5（邊緣更寬，暖金色光暈溢出更多） |
| 粒子場密度 | 降至 40%（機庫內部，粒子更稀薄，像機庫封閉空間） |
| 粒子色調 | 轉為以琥珀為主：70% 琥珀 `#D4940A`（透明 0.3–0.5），30% 近白（透明 0.2） |
| 粒子漂移速度 | 加速至 0.018（機庫內空氣流動）並加入輕微 Y 軸上下抖動（±0.002） |
| 環境光 | color 從 `#1A2A4A` 偏移至 `#1A1A0A`（冷藍轉暖黑），intensity 0.22 維持 |
| 地面光暈 | 強度升至 `rgba(212,148,10,0.18)`（琥珀地面燈最亮，機庫地板燈感） |
| 整體暗房強度 | 最深（最封閉），前景的 Fresnel 琥珀光與極暗背景形成最強對比 |

---

#### SCENE_ENTERPRISE（71%–90%，企業核心）

**氛圍語意**：轉向亞太俯視，地球靜止，展示全球佈局。是五個場景中最「沉著」的一個，企業感對應克制的靜態光源。

| 維度 | 規格 |
|------|------|
| Fresnel shader 顏色 | 從琥珀 `#D4940A` 過渡回冷藍 `vec3(0.102, 0.267, 0.537)` — 接近 `--color-deep` 的藍白色 |
| Fresnel intensity | 從 1.6 降至 0.8（最弱，精密儀器的冷靜）  |
| Fresnel power | 從 2.5 升回 4.0（邊緣最銳，像精密雷射切割線） |
| 粒子場密度 | 升至 70%（企業場景需要更多存在感，但以靜態為主） |
| 粒子漂移速度 | 降至 0.004（幾乎靜止，像衛星圖的長時間曝光） |
| 粒子色調 | 回到 80% 近白（`#E8EDF2`，透明 0.3–0.6），20% 琥珀（透明 0.2，淡出） |
| 環境光 | intensity 從 0.22 升至 0.38（稍亮，展示企業數據需要更清晰） |
| 地面光暈 | 降至 `rgba(212,148,10,0.04)`（幾乎消失，焦點移向地球而非地面） |
| 整體暗房強度 | 中淺（背景略亮，讓企業 KPI 數字的可讀性提升） |

---

#### SCENE_TERMINAL（91%–100%，終端機洞察）

**氛圍語意**：退出宇宙，進入純數據空間。地球縮至右下角，背景變為最暗，大量粒子模擬資料流。這是最接近 emons.de 「遊戲 UI」感的場景。

| 維度 | 規格 |
|------|------|
| Fresnel shader 顏色 | `vec3(0.0, 0.784, 0.831)` — 回到 cyan 主色，但 intensity 極低 |
| Fresnel intensity | 0.5（地球退場，Fresnel 應隱入背景，不再搶焦點） |
| Fresnel power | 5.0（極銳，細線感，地球在遠景中只剩一條精細藍光邊緣） |
| 粒子場密度 | 升至 200%（基準的兩倍，資料流視覺，密集粒子模擬資料包傳輸） |
| 粒子大小 | 縮小至 0.4–0.8px（比其他場景更細，像終端機像素點） |
| 粒子漂移速度 | 加速至 0.025，且增加強烈的 Y 軸向下漂移（0.012，模擬資料向下流動） |
| 粒子色調 | 60% 霓虹青 `#00C8D4`（透明 0.4–0.7），30% 近白（透明 0.3），10% `#20E0EC` cyan-glow |
| 環境光 | intensity 降至 0.18（最暗，逼近深空） |
| 地面光暈 | 消失（設為 0），背景純粹依賴粒子發光 |
| 整體暗房強度 | 最深（與 SCENE_SKUNKWORKS 同深，但氛圍是冷靜而非工業熱度） |

### A.2 場景切換光感過渡曲線

切換窗口為 600–1000ms，依場景組合使用不同曲線。

**切換時長對照**：

| 場景組合 | 切換時長 | 語意原因 |
|---------|---------|---------|
| IDLE → APPROACH | 800ms | 緩慢降低高度，不急躁 |
| APPROACH → SKUNKWORKS | 700ms | 解鎖動作，有力但不拖沓 |
| SKUNKWORKS → ENTERPRISE | 900ms | 轉換視角距離最遠（俯視 → 亞太），需要更長的鏡頭弧線 |
| ENTERPRISE → TERMINAL | 600ms | 退出動作，快速拉遠像「抽身離場」 |
| 任何場景 → 向上回退 | 同進場時長 | 回退速度等於進場，維持平衡（UX 規格） |

**Fresnel 屬性插值曲線（語意描述）**：

- Fresnel intensity 變化：全程使用 `ease-in-out`（前 30% 慢慢離開舊值，中間 40% 加速，後 30% 慢慢落定新值）。不使用 `linear`——線性的 Fresnel 變化會讓光感像在切換開關，而非有機呼吸。
- Fresnel color 插值：在 Three.js 中用 `THREE.Color.lerp()`，搭配 GSAP 的 `onUpdate` callback 逐幀更新 uniform，插值曲線使用 `power2.inOut`（等效 ease-in-out）。
- Fresnel power 變化：使用 `ease-out`（power 的增減是「聚焦」感，快速到位更有決斷性）。

**環境光 intensity 插值曲線**：

- 向暗過渡（intensity 降低）：`ease-in`（快速暗化，緊張感）
- 向亮過渡（intensity 升高）：`ease-out`（緩緩揭露，安靜感）

**粒子場密度過渡**：

粒子不直接增減數量（影響效能），改為：
- 向下密度：粒子群體的整體 `opacity` 透明度降低，配合 `ease-in`，300ms 完成（比光感早結束 100–200ms）
- 向上密度：透明度升高，配合 `ease-out`，在光感過渡的後半段才啟動（delay 200ms），製造「光先到，粒子後跟上」的層次感

**prefers-reduced-motion 靜態替代**：
場景切換時所有 Fresnel 漸變和粒子透明度過渡均停用（直接跳至目標值，無過渡動畫）。Three.js 的 uniform 以單幀更新。粒子場保持靜止（`opacity` 靜態設定，不做透明度動畫）。

## B. 解鎖動作的視覺反饋素材規格

### B.1 概念前提

UX 定義了「浮動卡片解鎖」的三步驟流程（ux-addendum-emons.md B.2 節），UI 定義了 Drawer 的進出動畫與 Skeleton（ui-addendum-emons.md B.2 節）。美術層要讓這三個步驟的「儀式感」真實可見——訪客要感受到「一份密件被解密」。

所有效果嚴格使用 `--color-accent #0EB8D5`（霓虹青）和 `--color-primary #D4920A`（琥珀金）的主色系，不引入任何新色彩。

---

### B.2 掃描線動效規格

**語意**：任務書滑入時，面板頂部有一條掃描線由上至下掠過，像機密文件被讀取機掃描解密。

| 屬性 | 規格 |
|------|------|
| 顏色 | `rgba(14,184,213,0.50)`——`--color-accent` 50% 透明（半透不刺眼，保留掃描感） |
| 寬度 | 100%（面板全寬） |
| 高度 | 2px |
| 速度 | 從面板頂部到底部，160ms（面板 Drawer 高度 ~80dvh，速度感適中） |
| 方向 | 單向，由上至下，只掃一次（非循環） |
| 觸發時機 | Drawer `.is-open` 狀態確認後，delay 20ms（確保 Drawer 邊框已完全出現） |
| 消退方式 | 掃描線到達底部後，`opacity: 0.50 → 0.00`，80ms ease-out（漸隱而非突然消失） |
| 實作方式 | `position: absolute`，`will-change: transform`，用 `translateY` 從 0 推至面板高度，單次動畫不循環 |

**掃描線後殘光（尾跡效果）**：

掃描線後方 8px 內有一條漸淡的殘光尾跡：
- 用 CSS `box-shadow` 製作：`0 0 8px 2px rgba(14,184,213,0.15)`
- 隨掃描線一同移動，不單獨動畫
- 掃描線消退時同步消退

**prefers-reduced-motion 替代**：掃描線動畫整體停用，Drawer 滑入後面板頂部出現一次靜態 2px 青色線條（`opacity: 0.30`，1 秒後 opacity 過渡至 0）。

### B.3 閃爍節奏規格

**語意**：系統確認解鎖的視覺確認信號，像雷達收到回波後的提示閃爍。

#### B.3.1 左欄指示燈解鎖閃爍

觸發條件：使用者點下左欄列表項後的「確認選中態」（0–150ms 窗口，ux-addendum-emons.md B.2 步驟 1）。

| 屬性 | 規格 |
|------|------|
| 閃爍目標 | 被選中產品的側邊狀態指示燈（●） |
| 閃爍節奏 | 3 下，每下 50ms 亮 / 50ms 暗，共 300ms（與 UX 規格一致） |
| 頻率 | 約 10Hz（50ms 週期），接近航空 ATIS 警告閃爍頻率 |
| 亮態顏色 | 對應狀態色（`--color-status-active`、`--color-status-approach` 等），`filter: brightness(2.0)` |
| 暗態顏色 | 對應狀態色，`filter: brightness(0.3)`（不熄滅，保留辨識性） |
| 持續時間 | 僅在點擊確認的 300ms 內，之後恢復對應狀態燈的常亮值 |

#### B.3.2 HUD 底部警示閃爍

觸發條件：地球鏡頭移動中（步驟 2，150–950ms）。

| 屬性 | 規格 |
|------|------|
| 閃爍目標 | 底部 HUD 的「FLIGHT QUEUE」數值 |
| 閃爍節奏 | 持續閃爍，200ms 亮 / 200ms 暗，5Hz |
| 亮態顏色 | `--color-accent`（`#0EB8D5`） |
| 暗態顏色 | `--color-foreground-muted`（`#7A8FA8`）——不完全熄滅 |
| 持續時間 | 鏡頭移動全程（最多 800ms），鏡頭就位後立即停止 |

#### B.3.3 任務書面板標題閃爍

觸發條件：Drawer 完全展開後，「MISSION UNLOCKED」文字出現（步驟 3，950–1200ms）。

| 屬性 | 規格 |
|------|------|
| 閃爍目標 | 「MISSION UNLOCKED」文字行 |
| 閃爍節奏 | 2 下快速閃爍，每下 120ms 亮 / 80ms 暗，之後保持常亮 |
| 頻率 | 約 5Hz，配合確認感而非緊張感 |
| 亮態顏色 | `--color-primary`（`#D4920A`，琥珀金，代表「任務啟動」的 B2B 層級色） |
| 常亮後顏色 | 同 `--color-primary` |
| 消失方式 | 1500ms 後 `opacity: 1.0 → 0.0`，300ms ease-out |

**prefers-reduced-motion 替代**：所有閃爍動畫停用。指示燈保持常亮；HUD 文字改為靜態顏色切換（一次，無動畫）；「MISSION UNLOCKED」直接顯示後 2 秒消失（opacity 切換，無過渡）。

### B.4 光暈擴散規格

**語意**：點擊觸發解鎖的瞬間，要有一個「能量從點擊位置向外擴散」的確認感，像在科幻電影中按下啟動按鈕。

**擴散起點**：從被點擊的左欄列表項位置（而非面板邊緣）。具體起點為列表項的中心點，座標由 JS `getBoundingClientRect()` 取得。

| 屬性 | 規格 |
|------|------|
| 擴散形狀 | 橢圓（非正圓），寬高比 3:1（水平扁橢圓，呼應 HUD 的水平儀表板感） |
| 起始尺寸 | 0px × 0px |
| 終止尺寸 | 寬 240px × 高 80px |
| 擴散時長 | 400ms |
| 擴散曲線 | `cubic-bezier(0.0, 0.0, 0.2, 1.0)`——極快啟動，緩慢落定（爆發感而非緩進） |
| 顏色 | `--color-accent`（`#0EB8D5`） |
| 透明度曲線 | 起始 `opacity: 0.25`，在 200ms 時達到峰值 `opacity: 0.35`，400ms 結束時回落至 `0.00` |
| 混合模式 | `mix-blend-mode: screen`（疊加在深色背景上，不遮蔽下方文字） |
| z-index | 比列表項高 1 層，比 Peek Panel 低 |
| 觸發次數 | 單次，不循環 |

**實作方式**：`position: fixed` 的絕對定位層，以 `transform: scale()` 從 0 放大，`border-radius: 50%` 製作橢圓，`pointer-events: none`。

**prefers-reduced-motion 替代**：光暈擴散完全停用，不顯示任何光暈。

### B.5 打字機效果參數規格

**語意**：Drawer 面板中「MISSION NOTES」和底部 HUD 的訊息更新，以打字機效果顯示，強化「資料灌入系統」的即時感。不同文字區塊使用不同的字符集和速度。

#### B.5.1 任務書標題（「MISSION UNLOCKED」文字）

| 屬性 | 規格 |
|------|------|
| 每字符延遲 | 40ms（共 16 個字符，總顯示時長約 640ms） |
| 字符集（亂碼期） | 僅使用大寫英文字母 `A-Z`（不含數字，保持專業感） |
| 亂碼持續時間 | 每個字符在定格前顯示 2 個隨機字符，各 20ms（亂碼期：40ms，定格：持續） |
| 滑鼠懸停效果 | 懸停於此行時，文字再次觸發一次亂碼循環（單次，120ms 完成） |

#### B.5.2 Drawer 面板的 MISSION NOTES 內文

| 屬性 | 規格 |
|------|------|
| 每字符延遲 | 18ms（中文字符）/ 12ms（英文和符號） |
| 最大顯示時長上限 | 1200ms（超過則跳過剩餘字符直接顯示完整文字，防止過長等待） |
| 字符集（亂碼期） | 不使用亂碼（純打字機，直接逐字顯示）——中文字符的亂碼缺乏語意，避免混亂感 |
| 光標 | 末尾顯示 `▋`（全形方塊），顏色 `--color-accent`，800ms 閃爍週期，文字完成後持續 1.5 秒後消失 |

#### B.5.3 底部 HUD 的 Text Scramble 效果（更新既有數值）

配合 art-direction.md 9.2 節「Text Scramble Effect」，補充具體參數：

| 屬性 | 規格 |
|------|------|
| 每字符延遲 | 30ms |
| 字符集（亂碼期） | 僅使用數字 `0-9`（HUD 顯示的是數值，亂碼也應是數字，保持儀表板感） |
| 亂碼輪次 | 每個字符在定格前顯示 3 個隨機數字，各 10ms（亂碼期：30ms，定格：持續） |
| 觸發週期 | 6–10 秒隨機（與 art-direction.md 8.2 節 HUD 文字動畫規格一致） |
| 更新時文字顏色 | 亂碼期間：`--color-muted`（`#7A8FA8`）；定格後：恢復原本的 `--color-accent` / `--color-foreground` |

**prefers-reduced-motion 替代**：所有打字機效果停用，文字直接完整顯示。HUD 數值更新直接切換（無亂碼過程）。

## C. 遊戲感氛圍細節清單

### C.1 音頻隱喻（動效設計中的「聲感」）

沒有實際聲音輸出，但動效必須讓訪客感知到聲音的存在，如同觀看無聲電影時腦補音效的體驗。每個互動事件對應一個「聲感動效語意」。

| 互動事件 | 聲感語意 | 對應動效實現 |
|---------|---------|------------|
| 點擊左欄列表項 | 短促電子確認音（「bip」，如機場登機閘門確認） | 指示燈 3 次快閃（50ms 週期）+ 光暈橢圓爆散（400ms） |
| 地球鏡頭開始移動 | 低頻機械旋轉聲（「whirr」，如衛星天線轉向） | 大氣層 Fresnel intensity 瞬間提升 0.2（100ms），搭配粒子場輕微加速 |
| Drawer 滑入 | 抽屜開啟聲（「schick」，精密機械感） | 掃描線從頂部高速掠過（160ms），backdrop-filter blur 從 0 快速展開 |
| 場景切換完成 | 低沉的「到達」音效（「thud」，如艙門鎖定） | Fresnel 強度在切換完成瞬間有一個 30ms 的輕微超出（overshoot），然後回落到目標值 |
| HUD 數值更新 | 電子嘀嘀聲（「beep」，儀表板刷新） | Text Scramble 的亂碼期顏色切換為 `--color-muted`，定格時顏色回亮（有對比感） |
| 雷達掃描線經過飛機 | 雷達偵測音（「ping」） | 飛機 sprite 亮度在掃描線路過的 200ms 窗口內提升至 1.4x，然後回落 |
| 滑鼠 hover 飛機 sprite | 微弱目標鎖定音（「tick」） | sprite 尺寸 1.0 → 1.2 的 150ms 放大，配合脈衝環轉為常亮（art-direction.md 8.3 節） |

**設計原則**：「聲感」的強弱依重要性分層。主要互動（點擊、場景切換）的動效要「響亮」；環境動效（雷達掃描、HUD 更新）要「低調」。訪客不應因為過多的視覺「聲響」而感到疲勞。

### C.2 「系統正在運作」的環境動態清單

以下元素需要持續微動，製造「活的系統」感知。幅度細節如下：

| 元素 | 動態類型 | 幅度 | 週期 | 說明 |
|------|---------|------|------|------|
| 地球自轉 | 連續旋轉 | 完整圓周 | 60 秒 / 圈 | 主要持續動態，所有場景均保持（SCENE_ENTERPRISE 除外，該場景靜止） |
| 大氣層 Fresnel 呼吸 | `emissiveIntensity` 微波動 | ±0.05（約 3.5% 的基準值） | 8 秒週期 | 感知需要專注觀察，只讓知道地球「活著」的人察覺（art-direction.md 8.2 節） |
| 雷達掃描環旋轉 | 連續旋轉 | 完整圓周 | 4 秒 / 圈 | 視覺上像衛星通訊站的掃描面，慢於真實雷達（製造沉靜感） |
| 飛機 sprite Bob 動畫 | `translateY` 上下浮動 | ±2px（場景單位 ±0.008） | 3 秒週期 | 模擬飛機在氣流中的輕顛，每架飛機的相位偏移 0.6–1.2 秒（避免同步顯得假） |
| 弧線粒子流 | 沿弧線路徑移動 | 單個粒子：完整弧線長度 | 4–8 秒 / 圈（依狀態） | 近景弧線（APPROACHING）速度較快 4 秒，遠景（HOLDING）較慢 8 秒 |
| HUD 底部數值更新 | Text Scramble（見 B.5.3 節） | 數值差異 | 6–10 秒隨機 | CONNECTED NODES 和 FLIGHT QUEUE 數值動態刷新 |
| 左欄 HOLDING 狀態指示燈 | `opacity` 脈衝 | 0.3 → 1.0 → 0.3 | 2 秒週期 | 代表飛機在盤旋等待，慢速表示「不急但在等」 |
| 左欄 ON FINAL 狀態指示燈 | `opacity` 快速閃爍 | 0.2 → 1.0 → 0.2 | 0.6 秒週期 | 模擬急迫進場的 NOTAM 警示（art-direction.md 7.2 節） |
| 底部 HUD 的 SYSTEM HEALTH 指示燈 | `box-shadow` 呼吸 | `0 0 4px 2px rgba(34,197,94,0.40)` → `0 0 8px 4px rgba(34,197,94,0.60)` → 回落 | 2 秒週期 | 深綠指示燈的光暈呼吸，系統健康感 |
| 背景環境粒子場漂移 | 三維緩慢漂移 | 每幀 0.008 場景單位（SCENE_IDLE 基準） | 無週期（線性漂移） | 最不顯眼的持續動態，製造空間深度感 |

**效能預算備注**：以上 10 個持續動態元素，除 Three.js 的前 4 個外，其餘均使用 CSS `@keyframes` 或 CSS `animation`，不消耗 Three.js 的渲染資源。在低效能裝置（`prefers-reduced-motion: reduce` 或 fps < 30 檢測）時，以下項目優先停用：
1. 飛機 Bob 動畫（視覺損失最小）
2. 大氣層呼吸（幅度極小，停用感知差異不大）
3. 背景環境粒子場漂移（改為靜態，不移動）

### C.3 危機感元素：HUD 警報閃爍規格

「危機感」是 emons.de 遊戲感的核心——玩家需要感覺到「有事情正在發生」才會保持注意力。V3 在以下特定條件下讓 HUD 數值進入警報狀態，製造輕微的壓力感，但不讓訪客感到不安。

---

#### C.3.1 觸發條件與視覺規格

| 觸發條件 | 警報目標 | 視覺規格 | 持續時間 |
|---------|---------|---------|---------|
| 飛機狀態為 ON FINAL（一架或多架正在進場） | 左欄「ON FINAL」列表項的右側新增警示標籤「FINAL」 | `--color-status-final`（`#E4572E`），JetBrains Mono 9px，大寫，`opacity: 1.0 → 0.3` 閃爍，1.0 秒週期 | 該飛機維持 ON FINAL 狀態期間持續 |
| 飛機狀態為 ON FINAL | 底部 HUD 的 FLIGHT QUEUE 數值 | 數值顏色從 `--color-foreground` 切換至 `--color-status-final`（`#E4572E`），加 `text-shadow: 0 0 6px rgba(228,87,46,0.40)` | 同上 |
| scroll 進入 SCENE_TERMINAL（91% 以上） | 底部 HUD 整列 | HUD 背景疊加 `rgba(0,200,212,0.03)` 底染（極微弱青色，暗示進入資料空間），`SYSTEM HEALTH` 文字從「OPTIMAL」替換為「TERMINAL MODE」 | SCENE_TERMINAL 期間持續 |
| 地球鏡頭移動中（凍結狀態） | 底部 HUD 的 SYSTEM HEALTH 文字 | 替換為「PROCESSING: CAMERA LOCK」，顏色 `--color-primary`（`#D4920A`，琥珀），無閃爍（沉穩處理感） | 鏡頭移動完成前（最多 800ms）|
| 使用者在頁面停留超過 90 秒但未與任何產品互動 | 底部 HUD 整列，輕微一次性提示 | 新增「AWAITING INPUT...」字串從右側打字機出現，`--color-foreground-muted` 色，15 秒後自動消失（不重複觸發） | 15 秒，單次 |

---

#### C.3.2 警報閃爍視覺參數

**ON FINAL 狀態警示閃爍（主要警報）**：

| 屬性 | 規格 |
|------|------|
| 閃爍頻率 | 1.0Hz（`opacity: 1.0 → 0.3`，500ms 亮 / 500ms 暗） |
| 閃爍顏色 | `--color-status-final`（`#E4572E`）——橘紅色，醒目但不是純紅色（純紅過於恐慌） |
| 發光半徑 | `text-shadow: 0 0 6px rgba(228,87,46,0.40)` |
| 伴隨效果 | 指示燈同頻閃爍（0.6 秒週期，比警報文字快，製造雙頻的複雜感知） |
| 影響範圍 | 僅限 ON FINAL 狀態的列表項和底部 HUD 的 FLIGHT QUEUE——不影響其他 UI 元素 |

**輕微系統提示閃爍（低優先級）**：

在環境動態已定義的持續閃爍之外，以下情況觸發額外的一次性輕微閃爍：

| 觸發 | 目標 | 閃爍規格 |
|------|------|---------|
| 進入任何新場景（scroll snap point 確認） | 底部 HUD 的場景名稱欄（`SECTOR: XXX`） | 2 次快速閃爍，200ms 週期，`--color-accent` 顏色，閃後恢復 |
| Drawer 成功解鎖並完全展開 | 頂部 Nav 的品牌標識（雙環同心圓，art-direction.md 7.2 節） | 外環虛線出現一次完整旋轉（360 度，400ms ease-out），旋轉後恢復靜態 |
| 使用者完整閱讀任務書後點擊主要 CTA | 底部 HUD 的 FLIGHT QUEUE 數值 | 數值 -1（一架飛機「已著陸」），Text Scramble 更新，配合一次 `--color-status-active`（綠色）短暫色彩切換 |

---

#### C.3.3 警報視覺的克制原則

**最多同時出現 1 個警報類動畫**。若 ON FINAL 警報正在進行，其他輕微閃爍排隊等待（queue），不疊加。

**警報的「音頻隱喻強度」分級**：

| 等級 | 觸發條件 | 視覺強度 | 對應聲感 |
|------|---------|---------|---------|
| Level 1（環境提示） | HUD 數值更新、場景切換確認 | 極輕微，1–2 次閃爍 | 輕柔的儀表板 beep |
| Level 2（狀態警示） | ON FINAL 狀態持續 | 持續慢閃，不誇張 | 柔和的進場警告音 |
| Level 3（系統事件） | Drawer 解鎖、地球鏡頭移動 | 明確的一次性效果（掃描線、光暈擴散） | 機械解鎖聲、衛星旋轉聲 |

Level 3 的效果最強烈，但也是最稀少的——只在使用者主動觸發互動時出現，不在環境層面持續。

**prefers-reduced-motion 替代**：所有警報閃爍和持續脈衝動畫停用。ON FINAL 狀態改為靜態顏色標識（`--color-status-final` 常亮，無閃爍）。HUD 狀態替換為直接文字更新（無動畫過渡）。

## D. prefers-reduced-motion 完整靜態替代方案總覽

以下為本文件涉及的所有動效的靜態替代方案彙總，工程師實作時以此清單為檢查依據。

| 動效 | 靜態替代 | 實作方式 |
|------|---------|---------|
| Fresnel 顏色過渡（場景切換） | 直接跳至目標值，無漸變 | `prefers-reduced-motion` media query 內 Three.js uniform 單幀更新 |
| 粒子場透明度過渡（場景切換） | 直接切換目標透明度 | CSS class 切換，無 transition |
| 掃描線動效（Drawer 開啟） | 靜態 2px 青色頂部線條，1 秒後 opacity 0 | CSS class 切換，transition 保留（功能性） |
| 指示燈閃爍（點擊確認） | 指示燈常亮（對應狀態色） | 移除 `animation` property |
| HUD 閃爍警報 | 靜態顏色標識（`--color-status-final` 常亮） | 移除 `animation` property |
| 光暈橢圓擴散（點擊確認） | 完全停用，不顯示 | 移除觸發 JS 邏輯 |
| 打字機效果 | 直接顯示完整文字 | 移除逐字符延遲 JS 邏輯 |
| Text Scramble（HUD 更新） | 直接顯示新數值，無亂碼過程 | 移除亂碼字符循環 |
| 大氣層呼吸（Fresnel 微波動） | 靜態 Fresnel intensity，不波動 | 移除 GSAP tween |
| 飛機 Bob 動畫 | 飛機 sprite 靜止 | 移除 `translateY` tween |
| 雷達掃描環旋轉 | 靜態圓環，不旋轉 | 移除旋轉 tween |
| 背景環境粒子場漂移 | 靜態粒子，不漂移 | 移除粒子位移 update |
| 品牌標識旋轉（Drawer 解鎖） | 靜態外環 | 移除旋轉 animation |

```css
/* 統一 prefers-reduced-motion 處理入口 */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }

  /* Drawer 滑入保留（功能性位移，使用者需看到面板從哪裡來） */
  .intel-drawer {
    transition-duration: 220ms !important;   /* 恢復 Drawer 的 transition */
  }

  /* 底部 Sheet 滑入保留（同理） */
  .product-sheet {
    transition-duration: 300ms !important;
  }
}
```
