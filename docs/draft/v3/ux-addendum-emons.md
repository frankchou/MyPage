## A. Gemini 骨幹（Skeleton）保留規格

### A.1 骨幹定義與設計原則

V3 的設計原則是「骨幹不動，皮膚可換」。Gemini 示意圖提供了一個在 portfolio 設計中極為稀有的三欄 ATC 架構，這個架構的空間邏輯是整個「航管中心」體感的基礎。任何再設計必須在此骨幹之內進行，而非破壞它。

骨幹的核心邏輯：
- 三欄分工是「職能」不是「裝飾」：左欄 = 指令輸入（用戶選擇產品），中欄 = 執行中心（地球視覺），右欄 = 情報輸出（數據回饋）
- 頂部和底部的 HUD 條形成閉合的「指揮中心介面框架」，讓整個頁面像一個系統控制台
- 移除任何一個邊界元素，閉合感消失，航管中心體感瓦解

---

### A.2 鎖死不可改的結構元素

以下結構為骨幹核心，任何迭代設計均不得移除、合併或根本性重組：

#### 頂部導覽列（Top Navigation HUD）
- 必須存在：品牌標識區（左）+ 頁面名稱區（中）+ 功能入口區（右）
- 必須是固定（sticky）的全寬條形元素，高度上限 80px
- 必須與底部 HUD 在視覺語言上形成呼應（等寬字體、系統感邊框）
- 不可改的原因：沒有頂部 HUD，介面看起來像普通網站而非控制台

#### 三欄非對稱比例
- 三欄必須同時存在於 1024px 以上斷點（不可合併為二欄）
- 比例鎖定範圍：左欄 18-22% / 中欄 50-58% / 右欄 22-28%（可在此範圍內微調）
- 中欄必須是視覺最重的欄（地球儀所在），不可讓左欄或右欄在視覺上壓過中欄
- 不可改的原因：比例失衡會破壞「地球是指揮中心」的空間感

#### 底部 HUD 狀態列（Bottom Status Bar）
- 必須存在：系統狀態數值（至少 3 個數值欄位）
- 必須是全寬固定（sticky）或頁面底部吸附的條形元素
- 必須使用等寬字體（Mono），視覺語言與頂部 HUD 呼應
- 不可改的原因：底部 HUD 是「系統正在運作」活躍感的錨點，移除後頁面變成靜態展示

#### 中欄的地球儀語意
- 中欄的主視覺主體必須是地球（3D 或降級的 2D SVG），不可替換為其他主視覺
- 地球必須是可互動的（可旋轉，或在降級方案中有動態效果）
- 不可改的原因：地球儀是整個「全球指揮官」隱喻的具象化，替換任何元素都會讓隱喻崩潰

#### 左欄的進場航班列表語意
- 左欄必須維持「列表型態」的 AI 產品展示，每個項目可點擊
- 必須保留狀態標籤（APPROACHING / HOLDING / ON FINAL / TAXIING / ACTIVE），或等效的航空術語系統
- 不可改的原因：列表點擊觸發地球聯動是核心互動的入口，列表語意消失則互動鏈斷裂

### A.3 可再設計的元素（皮膚層）

以下元素在骨幹確認後可由 UI 設計師和美術設計師自由再設計：

| 元素 | 可再設計的範疇 |
|------|-------------|
| 視覺語言 | 色彩系統、光暈效果、毛玻璃程度、邊框樣式 |
| 字體選擇 | 在禁止清單外的任何字體組合 |
| 動畫曲線 | 過場動畫的 easing、飛行弧線的顏色與粗細 |
| 右欄模塊順序 | B2B KPI / 認證 / 地圖 / Tech Stack 的排列順序 |
| 左欄列表卡片 | 卡片高度、側邊燈顏色、hover 展開行為 |
| 飛行數據卡樣式 | 卡片形狀、資訊欄位的視覺呈現方式 |
| Hero 文字位置 | 在頂部導覽列下方和地球儀上方之間的任意位置 |
| 背景環境 | 夜間機場照片的裁切方式、疊加效果、色調處理 |
| 粒子場 | 密度、顏色、速度（在效能預算內） |

## B. emons.de Scroll-Driven 互動流程

以下 4 個互動機制以 emons.de 的「帶入式遊戲感、沉浸氛圍感」為靈感，在 V3 的骨幹框架內實作。

---

### B.1 Scroll-Driven 場景切換

#### 概念說明

emons.de 的 scroll-snap 讓使用者感覺在「推進關卡」而非瀏覽頁面。V3 的對應機制是：scroll 位置觸發 Three.js 場景狀態的轉換，每個場景狀態對應一個「主題區塊」（首頁 Command Center、AI 機庫、企業核心、終端機洞察）。

使用者在中欄向下捲動時，地球儀的鏡頭、光線、粒子密度會一起過渡到下一個場景，產生「進入不同太空站艙間」的空間感。

#### Scroll Threshold 定義

```
Scroll Position     場景狀態                  Three.js 變化
────────────────    ──────────────────────    ──────────────────────────────────────
0% – 15%            SCENE_IDLE                地球儀正常自動旋轉，5 架飛機進場動畫
                    （初始場景，Command Center） 鏡頭：正面偏右，略偏上

16% – 45%           SCENE_APPROACH            鏡頭緩慢推近地球（FOV 縮小 15%）
                    （進入機庫過場）            飛機弧線逐漸收縮到近地軌道
                                              粒子場密度降低 40%

46% – 70%           SCENE_SKUNKWORKS          鏡頭拉遠至地球右上角俯視角
                    （AI 機庫）               地球旋轉速度加快 50%
                                              弧線顏色從青色過渡至琥珀金色

71% – 90%           SCENE_ENTERPRISE          鏡頭水平移至正南視角（俯瞰亞太）
                    （企業核心）              地球停止自動旋轉（靜態聚焦）
                                              弧線從點狀連接切換為射線型

91% – 100%          SCENE_TERMINAL            鏡頭拉至最遠，地球縮小至右下角
                    （終端機洞察）             粒子場加密（模擬資料流）
                                              地球表面光線變暗
```

#### Snap Points

Scroll-snap 僅在以下閾值的 ±3% 範圍內啟動，避免強制卡格讓使用者感覺失控：

| Snap Point | 對應場景入口 | 說明 |
|-----------|-----------|------|
| 0% | SCENE_IDLE | 首頁入口，自動吸附 |
| 46% | SCENE_SKUNKWORKS | 進入機庫，輕微吸附（阻力感） |
| 71% | SCENE_ENTERPRISE | 進入企業核心，輕微吸附 |
| 91% | SCENE_TERMINAL | 終端機入口，輕微吸附 |

設計決策：不使用強制 scroll-snap（`scroll-snap-type: mandatory`），採用輔助式吸附（`scroll-snap-type: y proximity`），讓使用者保有自主掌控感，避免強迫症式卡格。

#### 正常流程（Happy Path）

```
使用者狀態：桌機，Three.js 已初始化，慢速向下捲動

1. 進入頁面
   → SCENE_IDLE 啟動
   → 地球自動慢速旋轉，5 架飛機開始動畫
   → 底部 HUD 數值開始閃爍更新

2. 向下捲動至 16%
   → 進入 SCENE_APPROACH 過渡
   → Three.js 鏡頭開始推近（GSAP tween，1000ms ease-out）
   → 飛機弧線動畫速度減半（像準備降落）
   → 左欄列表輕微上移淡出（30% 不透明度）

3. 繼續捲動至 46%
   → 進入 SCENE_SKUNKWORKS
   → 鏡頭完成推近，開始向右上角旋轉
   → 左欄內容切換為 AI 機庫產品列表（淡入淡出，200ms）
   → 右欄切換為機庫詳情面板
   → 如靠近 snap point：輕微吸附提示（卡格感停止約 0.3 秒）

4. 繼續捲動至 71%
   → 進入 SCENE_ENTERPRISE
   → 鏡頭轉至亞太俯視角，地球旋轉停止
   → 弧線圖案切換（琥珀金色射線連接至台灣節點）
   → 右欄顯示 B2B KPI 強化版（更大字級）

5. 捲動至 91%
   → 進入 SCENE_TERMINAL
   → 地球縮小，退至右下角
   → 左欄出現終端機文字列表
   → 粒子場密度提升，製造資料流視覺
```

#### 空狀態

不適用於 scroll 場景切換。每個場景狀態都有預設視覺內容，不存在「無資料」情況。

#### 載入狀態

```
觸發條件：Three.js 場景切換的 3D 資源尚未載入（首次進入某場景）

體驗設計：
- Three.js 場景切換為漸進式（不等待全部資源載入）
- 地球紋理使用漸進式貼圖：先顯示低解析度版（200KB），
  高解析度版載入完成後無縫替換（不閃爍）
- 弧線動畫在資源載入期間降級為 CSS SVG 版，Three.js 版本
  就緒後自動替換（使用 opacity 交叉淡入，300ms）
- 載入期間底部 HUD 顯示：「LOADING SECTOR DATA...」
  取代原本數值，閃爍動畫提示系統活躍

視覺回饋：
- 不使用 spinner
- 用底部 HUD 文字替換傳遞系統正在工作的感覺
- 整體場景仍可互動（地球仍可旋轉），只是部分元素降級
```

#### 錯誤狀態

```
觸發條件：Three.js 場景資源載入失敗（網路錯誤、WebGL 崩潰）

降級方案：
1. 輕度失敗（某場景資源 404）：
   - 當前場景保持上一場景的視覺狀態，不強制過渡
   - 底部 HUD：「SECTOR DATA UNAVAILABLE — MAINTAINING CURRENT BEARING」
   - 用戶可繼續捲動，但場景不會切換（UI 保持靜態）
   - 3 秒後自動重試（靜默）

2. 重度失敗（WebGL context 丟失）：
   - 整個 Three.js canvas 替換為靜態 PNG 背景（預渲染的地球圖）
   - 底部 HUD：「RADAR OFFLINE — MANUAL MODE ACTIVE」
   - Scroll-driven 場景切換改為純 CSS class 切換
     （背景色和左欄內容仍會隨 scroll 改變，只是沒有 3D 效果）
   - 右下角非侵入提示：「已切換至輕量模式」（5 秒後自動消失）

復原路徑：
- 不要求使用者手動重新整理
- WebGL context 若恢復（瀏覽器背景 tab 返回），自動重新初始化
```

#### 回退行為（Back-scroll）

```
向上捲動時，場景逆向過渡：

- SCENE_TERMINAL → SCENE_ENTERPRISE：鏡頭反向移動，地球重新放大
- SCENE_ENTERPRISE → SCENE_SKUNKWORKS：弧線顏色反向過渡
- SCENE_SKUNKWORKS → SCENE_APPROACH：鏡頭反推回
- SCENE_APPROACH → SCENE_IDLE：地球恢復初始旋轉速度

設計原則：
- 回退動畫速度與進場相同（不加速，避免眩暈感）
- 回退時 snap point 同樣有效（輕微吸附）
- 中途停止（在兩個場景閾值之間）：場景保持最近一次
  完成的狀態，不懸停在過渡中間值
```

### B.2 浮動卡片解鎖（Floating Card Unlock）

#### 概念說明

emons.de 的「點擊才展開更多資訊，像解鎖任務」互動，在 V3 中轉化為：點擊左欄 AI 產品 → 地球鏡頭聚焦 → 右欄「任務書」滑入。整個序列設計成一個連貫的 3 步驟「解鎖動畫」，讓使用者感覺在「啟動一個任務」。

#### 完整互動步驟序列

```
觸發入口（二選一）：
A. 點擊左欄 Flight Dispatch 的產品列表項
B. 點擊中欄地球儀上的飛機 Sprite

─────────────────────────────────────────────────────────
步驟 1：確認選中（0ms – 150ms）

  左欄：
  - 被點擊的列表項：背景亮度提升，左側指示燈由常亮變為快速
    閃爍（3 下，每下 50ms），然後保持常亮（比預設更亮）
  - 其他列表項：opacity 降至 50%
  - 觸覺回饋：如設備支援 Vibration API，輕微震動（30ms）

  中欄（Three.js）：
  - 對應飛機 Sprite 縮放至 1.2x（150ms ease-out）
  - 飛機 Sprite 出現白色外輪廓光暈（pulse）

  右欄：
  - 當前內容開始淡出（opacity: 1 → 0.3，200ms）

─────────────────────────────────────────────────────────
步驟 2：地球聚焦（150ms – 950ms）

  中欄（Three.js）：
  - 鏡頭開始移動：GSAP tween，800ms，ease-in-out
  - 目標角度：使對應飛機弧線的終點（地球著陸點）朝向正面
  - 地球自動旋轉暫停（防止旋轉和鏡頭移動疊加產生眩暈）
  - 弧線顏色：被選中的弧線亮度提升至 100%，其餘降至 20%
  - 弧線動畫速度：選中弧線加速至 3x（像飛機加速進場）

  視覺提示：
  - 進度指示：底部 HUD 的 FLIGHT QUEUE 數值閃爍，顯示
    「FLT-0X APPROACHING...」（4 個字元追加動畫，打字機效果）

─────────────────────────────────────────────────────────
步驟 3：任務書解鎖（950ms – 1200ms）

  右欄：
  - 舊內容完全淡出（opacity: 0，完成時機：950ms）
  - 新的產品詳情面板從右側滑入（translateX: 40px → 0）
    同時 opacity: 0 → 1，時長 250ms，ease-out
  - 面板頂部顯示：「MISSION UNLOCKED」閃爍文字（顯示 1.5 秒後消失）
  - 產品資訊依序出現（stagger）：
    ① 產品名稱 + 狀態標籤（0ms）
    ② 飛行數據卡資訊（100ms 後）
    ③ 技術架構摘要（200ms 後）
    ④ 行動按鈕（300ms 後）

  左欄：
  - 被選中項目左側出現 2px 琥珀金線條（確認選中態）
  - 其他項目 opacity 回升至 80%

  地球：
  - 鏡頭到達目標角度後，恢復緩慢自動旋轉（速度降為原本 30%）
```

#### 狀態圖（文字版）

```
[預設態]
  左欄：5 個列表項，全亮，無選中
  中欄：地球自動旋轉，5 架飛機動畫
  右欄：預設 Intel Panel（B2B KPI + 認證 + 地圖 + Tech Stack）

  ↓ 使用者點擊任一產品

[確認選中態] （0-150ms）
  左欄：被選中項閃爍，其餘變暗
  中欄：對應飛機 Sprite 放大
  右欄：開始淡出

  ↓ 150ms 後自動進入

[地球聚焦態] （150-950ms）
  左欄：維持選中閃爍
  中欄：鏡頭移動，弧線聚焦
  右欄：淡出完成，等待新內容

  ↓ 950ms 後

[任務書解鎖態] （950-1200ms）
  左欄：閃爍轉為常亮，琥珀金線條出現
  中欄：鏡頭就位，地球緩慢旋轉
  右欄：產品詳情面板滑入，資訊依序出現

  ↓ 進入穩定選中態

[穩定選中態]
  左欄：選中項有琥珀金線條，其餘項 80% 不透明
```

### B.3 空間感導航（Spatial Navigation）

#### 概念說明

emons.de 讓使用者感覺在「穿越」不同空間，而非翻頁。V3 的對應設計是：不同主題區域的切換，用 Three.js 鏡頭移動來替代傳統的頁面跳轉。使用者始終「在同一個宇宙空間中」，只是鏡頭角度和場景狀態不同。

這個設計解決了一個關鍵問題：傳統 SPA 的路由切換，即使有過渡動畫，也有「瞬移」感。空間感導航讓使用者在移動過程中保持方位感——他們知道自己從哪裡來、要去哪裡。

#### 使用者方位感知系統

使用者需要能隨時回答「我在哪裡」，不可讓使用者迷失。以下機制共同建立方位感：

```
方位感知元素 1：底部 HUD 的場景標識

  SYSTEM HEALTH: OPTIMAL | [SECTOR: COMMAND CENTER] | UPTIME: 7Y
                            ↑ 當前場景名稱，隨鏡頭移動更新

方位感知元素 2：頂部導覽列的段落指示

  [COMMAND CENTER]  [AI SKUNKWORKS]  [ENTERPRISE CORE]  [TERMINAL]
        ●                 ○                  ○               ○
        ↑ 當前位置有實心圓點，其餘為空心

方位感知元素 3：地球儀上的方位指示針

  地球右上角顯示一個極小的羅盤玫瑰（Compass Rose），
  當鏡頭旋轉時，羅盤旋轉方向相反（保持方位固定），
  讓使用者理解自己在看地球的哪個角度。

方位感知元素 4：過渡動畫的方向性

  向右移動的頁面 = 向前（進入子主題）
  向左移動的頁面 = 向後（返回上層）
  向上移動 = 更宏觀（拉遠鏡頭）
  向下移動 = 更具體（推近鏡頭）
  這個方向性需要在所有導航動作中一致。
```

#### 鏡頭移動替代頁面跳轉的操作模型

```
傳統頁面跳轉               V3 空間感鏡頭移動
─────────────────────────────────────────────────────────
點擊「AI 機庫」導覽按鈕      點擊頂部導覽「AI SKUNKWORKS」
URL 改變 → 頁面重載          URL 改變（不重載）→ 鏡頭動畫

感受：消失，出現在新頁面       感受：飛越到新區域
方位感：完全喪失，在新地方     方位感：我從指揮中心飛到了機庫
返回方式：瀏覽器上一頁         返回方式：頂部導覽點擊 / 鏡頭返回動畫
```

#### 各導航節點的鏡頭路徑規格

```
Command Center → AI Skunkworks：
  鏡頭路徑：從正面偏右，向右下方推近（FOV 從 45° 縮至 35°）
  動畫時長：1200ms，ease-in-out
  中途視角：掠過地球表面（模擬飛機降落視角）
  URL：/ → /skunkworks（pushState，不觸發重載）

Command Center → Enterprise Core：
  鏡頭路徑：從正面偏右，旋轉至正南俯視（地球旋轉至台灣朝向正面）
  動畫時長：1500ms（距離較遠，時間稍長）
  中途視角：地球從斜面逐漸轉為俯視
  URL：/ → /enterprise

Command Center → Terminal：
  鏡頭路徑：鏡頭拉遠至最遠點，地球縮小至右下角
  動畫時長：900ms（拉遠感覺快，像「退出宇宙」）
  粒子場：拉遠過程中粒子場密度逐漸提升（資料化視覺）
  URL：/ → /terminal

任意頁面 → Command Center（返回主頁）：
  鏡頭路徑：返回初始角度（正面偏右，稍偏上）
  動畫時長：800ms，ease-out（返回比前進快）
  觸發方式：點擊頂部導覽左側品牌標識，或導覽項 [COMMAND CENTER]
```

#### 操作流程（正常路徑）

```
1. 使用者在 Command Center 閱讀

2. 點擊頂部導覽的「AI SKUNKWORKS」

3. 鏡頭開始移動（1200ms）
   → 頂部導覽：COMMAND CENTER 圓點消失，AI SKUNKWORKS 圓點閃爍（過渡中）
   → 底部 HUD：「NAVIGATING TO SECTOR: AI SKUNKWORKS」
   → 三欄主體：左欄和右欄內容淡出（opacity: 0.2）

4. 鏡頭到達目標角度（1200ms 後）
   → 頂部導覽：AI SKUNKWORKS 圓點變實心
   → 底部 HUD：恢復正常數值顯示，SECTOR 欄更新為「AI SKUNKWORKS」
   → 三欄左欄：切換為機庫產品列表（淡入）
   → 三欄右欄：切換為機庫詳情面板（淡入）
   → URL 已更新（使用者可複製分享此 URL）

5. 使用者在 AI SKUNKWORKS 互動

6. 按下瀏覽器返回（Back）
   → 觸發鏡頭返回動畫（800ms），回到 Command Center
   → URL 回退至 /
   → 行為等同步驟 4 的逆向
```

#### 空狀態

```
觸發條件：導航目標頁面的內容尚未設定（如 /terminal 目前無文章）

體驗設計：
- 鏡頭仍正常移動到目標角度（空間感不中斷）
- 到達後，左欄顯示空狀態 UI：
  「[SYS-001] DATABASE INITIALIZING」
  「情報庫建置中，首批資料預計 [日期] 解密」
  行動按鈕：「訂閱首批通知」
- 中欄地球儀：粒子場加密（暗示資料正在積累）
- 右欄：顯示預告卡（「即將揭露」類型的佔位內容）

方位感維持：
- 即使頁面內容為空，頂部導覽圓點和底部 HUD 的 SECTOR 標識仍正常更新
- 使用者知道自己在哪裡，只是這個地方目前空曠
```

#### 載入狀態

```
觸發條件：鏡頭移動期間，目標頁面的非同步資料正在請求

體驗設計：
- 鏡頭動畫不等待資料（立即開始移動）
- 鏡頭移動期間：左欄和右欄顯示骨架屏（鏡頭到達前先淡入骨架屏）
- 鏡頭到達目標後（1200ms）：
  - 若資料已回傳：骨架屏和真實內容交叉淡入
  - 若資料未回傳：骨架屏繼續顯示，shimmer 動畫表示等待中
- 底部 HUD：「LOADING SECTOR DATA... [進度條文字]」

設計原則：
- 鏡頭移動是主動作，資料載入是背景任務，不讓資料載入阻擋使用者的空間移動感
```

#### 錯誤狀態

```
觸發條件：導航目標頁面的資料載入失敗

體驗設計：
- 鏡頭仍完成移動（不因錯誤退回）
- 到達後，骨架屏替換為錯誤 UI：
  「[NAV-ERR] 無法取得情報資料」
  重試按鈕 + 返回主頁連結
- 底部 HUD：「SECTOR DATA TRANSMISSION FAILED — SIGNAL LOST」

復原路徑：
- 主要：「重試」按鈕，觸發資料重新請求（不觸發鏡頭動畫，只更新內容）
- 次要：「返回指揮中心」，觸發鏡頭返回 Command Center 動畫
- 降級：靜態緩存資料如有，顯示舊版本並標注「情報資料可能非最新版本」
```

### B.4 Mobile 降級（375px）

#### 設計原則

375px 下，Three.js 降級為 CSS/SVG 靜態或輕動畫版。設計目標不是「複製桌機體驗」，而是「在移除 3D 的限制下，保留相同的遊戲感和沉浸感」。三欄佈局收合為單欄垂直捲動，每個桌機互動機制都有對應的手機替代方案。

---

#### B.4.1 Scroll-Driven 場景切換的手機降級

```
桌機：Three.js 鏡頭移動 + scroll position 觸發場景轉換
手機替代：CSS class 切換 + CSS 背景漸變 + SVG 地球狀態切換

手機 Scroll 體驗流程：
────────────────────────────────────────────────────────
1. 進入頁面（0% scroll）
   → 頂部：Hero 標題 + 職銜
   → 中央：SVG 地球卡片（全寬）
     - 地球 SVG：顯示「SCENE_IDLE」狀態（飛行弧線 CSS 動畫）
   → 底部：B2B KPI 橫向 snap scroll 卡片

2. 捲動至 AI 機庫區段（IntersectionObserver 觸發）
   → SVG 地球卡片（若仍在視野中）：
     - CSS class 切換，弧線顏色從青色過渡至琥珀金
     - SVG 地球「旋轉感」CSS animation 加速
   → 背景色：深藍漸變至深琥珀（CSS transition，600ms）
   → 段落標題「[SECTOR: AI SKUNKWORKS]」淡入（打字機效果）

3. 捲動至企業核心區段
   → 背景色：切換至深綠系（企業感）
   → 段落標題更新

4. 捲動至終端機區段
   → 背景色：切換至深黑，粒子感 CSS 噪點紋理加強

降級的效果保留：
- 場景切換感：用背景色漸變和段落標題打字機動畫替代 3D 鏡頭移動
- 連貫性：使用者向下捲動時感覺在「進入」不同區域（而非純粹的頁面捲動）
- 方位感：頂部 sticky 導覽列有段落點指示（和桌機一致）
```

#### B.4.2 浮動卡片解鎖的手機降級

```
桌機：點擊左欄列表項 → 地球聚焦 → 右欄面板滑入
手機替代：手風琴 Accordion 展開 + 底部 Sheet 組件

手機解鎖流程：
────────────────────────────────────────────────────────
1. 使用者點擊 AI 產品列表項（手風琴標題列）

2. 確認選中（0-150ms）：
   - 列表項背景色亮度提升
   - 側邊彩色指示燈：快速閃爍 3 下（CSS animation）
   - 觸覺回饋：Vibration API，30ms

3. 手風琴展開（150-450ms）：
   - 列表項下方展開區域，height: 0 → auto
     （使用 CSS Grid row 技巧，transform: scaleY，不使用 height 動畫）
   - 展開內容依序淡入（stagger，與桌機面板一致的順序）：
     ① 狀態標籤 + 飛行代號（0ms）
     ② 飛行數據卡資訊（100ms）
     ③ 技術架構摘要（200ms）

4. SVG 地球儀（若在視野中）：
   - 對應弧線 CSS 動畫加速（替代 Three.js 聚焦效果）
   - 弧線顏色亮度提升

5. 「查看完整任務書」按鈕出現（350ms）：
   - 點擊 → 底部 Sheet 從底部滑入（translateY: 100% → 0）
   - Sheet 高度：75dvh
   - Sheet 內容：完整產品詳情（與桌機 Drawer 相同內容）

─────────────────────────────────────
手機空狀態：
  - 手風琴展開後，內容區顯示：
    「[CLASSIFIED] 任務書建置中」+ 「訂閱解密通知」按鈕
  - 行為與桌機空狀態一致

手機載入狀態：
  - 手風琴展開的高度動畫期間，內容區顯示骨架屏
  - 資料到達後，骨架屏和真實內容交叉淡入（200ms）

手機錯誤狀態：
  - 展開區域顯示：「讀取失敗，點擊重試」
  - 重試按鈕觸覺回饋（Vibration API，50ms）
  - 仍失敗：顯示靜態資訊 + 「前往完整頁面」外部連結
```

#### B.4.3 空間感導航的手機降級

```
桌機：Three.js 鏡頭移動替代頁面跳轉
手機替代：全螢幕展開選單 + 頁面平滑捲動 + 段落視覺切換

手機導航操作模型：
────────────────────────────────────────────────────────
主要導航：
- 頂部漢堡選單展開全螢幕選單
- 選單項目垂直排列，每項最小高度 56px
- 選單項目前有場景圓點（一致的視覺語言）

- 各主題區段以 IntersectionObserver 偵測進入視野
- 頂部 sticky 導覽列的段落圓點隨捲動更新（和桌機一致）

「穿越空間感」的手機替代方案：
- 全螢幕選單展開時：背景變黑，選單項目以 stagger 淡入
  （不是立即出現，像「系統加載頻道」）

  1. 選單項背景閃白（確認選中，100ms）
  2. 全螢幕選單淡出（200ms）
  3. 頁面平滑捲動至目標段落（behavior: smooth）
  4. 目標段落標題出現打字機動畫（強化「到達新區域」感）

方位感維持（手機版）：
- 頂部導覽列固定可見，段落圓點指示
- 底部 HUD（手機版縮減至 2 個數值）顯示當前 SECTOR
- 每個主題段落有清晰的全寬段落標題列（背景色輕微區隔）
```

#### B.4.4 觸控互動補充規格

```
手勢支援（iOS / Android 原生手勢）：

1. 手風琴開關：
   - 點擊（Tap）：展開/收合
   - 向左滑動（Swipe Left）：快速收合當前展開項
   - 不支援長按（避免與系統長按衝突）

2. 底部 Sheet：
   - 向下拖曳（Drag Down）：關閉 Sheet（iOS 原生 pattern）
   - 向上拖曳（Drag Up）：Sheet 高度從 75dvh 擴至 95dvh
   - 閾值：拖曳超過 Sheet 高度的 30% 才觸發關閉，防止誤操作

3. KPI 橫向捲動卡片：
   - 單指左右滑動
   - scroll-snap-type: x mandatory，確保每次滑動停在卡片邊緣

4. SVG 地球儀（輕量互動）：
   - 單指左右拖曳：地球 SVG 旋轉（2D 旋轉，CSS transform，不是 3D）
   - 不支援縮放（避免與頁面縮放衝突）

iOS 特殊處理：
- Safe Area Inset：底部 Sheet 底部 padding 加入 env(safe-area-inset-bottom)
- 底部 CTA 按鈕：同上
- min-height: 100dvh（避免 Safari 底部工具列跳動）
- -webkit-overflow-scrolling: touch（手風琴內容捲動）

Android 特殊處理：
- 使用 Pointer Events API 替代純 Touch Events（更廣泛相容性）
- 底部導覽列高度：Android 各廠商不同，統一使用 env(safe-area-inset-bottom) 處理

PWA 安裝模式（Standalone）：
- 移除 Safari / Chrome 瀏覽器 UI 後，頂部導覽列需加入 title bar drag region
  （讓使用者可以拖曳移動 PWA 視窗）
- 底部 CTA 的 safe area padding 仍需保留（部分 Android 設備有導覽條）
- 橫向旋轉（Landscape）：地球 SVG 卡片 max-height 設為 60dvh，
  KPI 卡片改為 2 列網格（不再橫向捲動）

prefers-reduced-motion（手機版）：
```
