## 核心美學決策（5 點）

**1. 移除照片背景，回歸純程式化介面**

Gemini 圖的機場跑道實拍照片是最根本的視覺問題。照片帶來光源衝突、LCP 效能負擔、版面不可控三個問題。改以 `#0a0a0f` 深曜黑 + CSS 點陣網格 + radial 環境光暈三層疊加，零圖片依賴，完全符合 PRD 的 LCP < 1.5s 要求，同時建立「航管中心暗室」的沉浸感基礎。

**2. Globe 從「地球儀」升級為「雷達任務空間」**

Gemini 的 3D 地球照片是「展示」，改進後的 SVG/Canvas Globe 是「操作介面」。網格線採 `--color-cyan` 5% 透明度，大陸輪廓幾乎隱沒，飛行弧線和移動光點才是視覺主角。使用者感受到的是「正在追蹤任務的雷達螢幕」，而不是「觀賞地球儀」。

**3. 強調色從 5 色收斂到 3 色語義系統**

Gemini 圖的 Globe 上同時出現霓虹青、紫色、綠色、粉色飛機圖示，Tech Stack 泡泡使用超出品牌色票的飽和紫色。v2 全面收斂到 v1 Design Token 的三色語義系統：`--color-cyan`（系統在線）、`--color-amber`（商業 B2B）、`--color-green`（AI ACTIVE 生命力）。每個顏色有唯一的語義角色，不混用。不新增任何顏色。

**4. 飛行弧線改為訊號傳輸語言，光點取代飛機圖示**

Gemini 圖使用截圖縮圖的飛機圖示，視覺噪音高、無法統一風格。改為 1.5px `stroke-dasharray: 8 4` 虛線弧線（封包傳輸感）+ 3–5px 程式化圓點（依狀態調整大小與 glow）。弧線的 dashoffset 流動動畫和光點的 animateMotion 行進，讓整個 Globe 從「靜態展示圖」變成「動態任務看板」。

**5. 視覺風格選型確認：Bioluminescent Solarpunk 延用 v1，不重新選型**

v2 是 v1 的視覺升級，不是全新設計方向。v1 選定的 Bioluminescent Solarpunk 在 v2 的 Globe 環境中得到更完整的體現：飛行弧線如生物發光觸手在暗海延伸，移動光點如深海生物訊號。這個隱喻比 v1 的純雷達元件更有生命力，且與 Gemini 圖的「飛機在地球上飛行」概念形成有意義的昇華，而非顛覆。

## 與 UI 設計師提案的互補關係

UI 設計師的 `ui-improvement.md` 專注於**版面結構與元件規格**（三欄寬度比例、status badge 規格、Design Token 定義、Globe 的 SVG 元件列表），美術設計師的 `art-direction.md` 專注於**視覺感受與氛圍語言**（為什麼這樣做，每個視覺選擇的情緒目標）。

| 面向 | UI 設計師（ui-improvement.md） | 美術設計師（art-direction.md） |
|------|-------------------------------|-------------------------------|
| 三欄佈局 | 24% / 52% / 24% 寬度比例規格 | 不重複；本文件聚焦各欄的視覺氛圍強化 |
| Globe | SVG 元件技術清單（經緯線、大陸 path、弧線、光點） | 顏色透明度規格、glow CSS 規格、為何這樣選色的設計理由 |
| 背景 | 確認底色為 `#080C10` | 確認背景三層疊加的完整 CSS 規格與光暈方向設計 |
| Design Token | 定義 5 個 Token 變數 | 確認所有顏色使用場景的語義角色與情緒目標 |
| 飛行弧線 | 確認使用 SVG quadratic bezier + stroke-dashoffset 動畫 | 規定 stroke-dasharray 模式、動畫速度、ACTIVE 狀態顏色切換邏輯 |
| 視覺風格 | 不涉及 | 從 22 種風格確認選型並說明理由 |
| 氛圍強化 | 不涉及 | 航管中心暗室四層強化策略（光源消除 / 發光層次 / 系統狀態語言 / 互動光效回饋） |

兩份文件合併參考，工程師可以得到完整的「規格（UI）+ 理由（Art）」雙軌指引，無資訊缺口。
