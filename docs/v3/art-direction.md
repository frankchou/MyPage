### 2.2 V3 如何在保留方向的同時超越預設

**清除 AI 紫色**
薰衣草紫是 AI 紫色系的衍生，在技術型 portfolio 中被過度使用。V3 的飛機圖示和弧線統一使用霓虹青（`#00C8D4`）。只有一個強調色體系，不引入第三顏色。

**分層毛玻璃**
建立毛玻璃密度三階層：
- Layer 0（地球 Canvas 背景）：無毛玻璃，純深色場景
- Layer 1（左右欄主面板）：輕度毛玻璃，`blur(8px)` + 10% 白色底
- Layer 2（飛行數據卡、Tooltip）：重度毛玻璃，`blur(20px)` + 15% 白色底，代表最前景資訊

**解除置中 Hero**
主標題移至左欄上方，以左對齊呈現。地球儀純作視覺重心，中央無疊加文字，回歸 VARIANCE:9 的非對稱美學。

**以精確數字替代填充值**
飛行數據卡改用真實對應：
- ALTITUDE = 產品版本號（`v2.1`）
- PASSENGERS = 活躍用戶數（真實或估算的不規則數字，如 `127 active`）
- TECH = 核心技術棧縮寫

**弧線精緻化**
弧線外發光從 `blur(8px)` 縮減至 `blur(3px)`，用粒子流（Three.js Points 沿 CatmullRomCurve3）替代粗糙發光，視覺上更像真實數據串流而非霓虹燈管。

不是科幻電影的霓虹都市，也不是 NASA 白色清潔室。是介於兩者之間：深夜的現實操作室，有精準的儀表板、有機場塔台窗外的跑道燈光，有衛星接收站的專注沉靜。

## 5. Three.js 地球視覺美學

### 5.1 地球材質感定位

**選型：「精密科學儀器」風格化寫實**

不選純寫實（NASA 衛星照）：照片質感與其他 UI 元素的扁平化精緻度脫節，且夜間模式下顏色太雜。

不選純全息/抽象（線框地球）：失去業主最喜愛的「真實地球上的飛行路徑」具象感，也失去地理意義。

選定方案：**夜間衛星地圖 + 程序化調色**，具體做法：
- 底層使用 NASA 公開的夜間地球光照貼圖（night earth texture），可清楚看到各大城市的燈光群落
- 疊加程序化噪點 shader，增加「訊號掃描」質感
- 整體色調往 hsl(210, 50%) 偏移，使地球的色溫與背景整體冷藍調一致
- 陸地邊界線用極細的青色線條強調（可選）

此方案的優勢：城市燈光在夜間地圖上天然對應業主的「跨國業務節點」，讓 Global Network 的概念在地球上直接可見，無需額外 overlay。

---

### 5.2 光源設定

```
// Three.js 光源配置

// 環境光：冷藍調，維持整體夜間氛圍
AmbientLight: color=#1a2a4a, intensity=0.3

// 主方向光（模擬太陽）：從左上 10-2 鐘方向射入
DirectionalLight: color=#2a4a8a, intensity=0.8
  position: { x: -2.5, y: 1.5, z: 2 }

// 大氣邊緣補光（模擬太空反照光）：從右下方
DirectionalLight: color=#0a1530, intensity=0.3
  position: { x: 2, y: -1, z: -1 }

// 地球本身的城市燈光：透過 emissiveMap + emissiveIntensity 模擬
  emissiveMap: nightEarthTexture
  emissiveIntensity: 0.6（夜面才發光，日面接近0）

// 點光源（地球指揮台氛圍燈，模擬塔台窗下的橘黃地面燈）
PointLight: color=#D4940A, intensity=0.2
  position: { x: 0, y: -3, z: 1 }  // 來自地球下方
```

### 5.3 大氣層效果

使用 GLSL Fresnel Shader 製作大氣層發光邊緣：

```glsl
// Fresnel 公式（簡化）
float fresnel = pow(1.0 - dot(vNormal, vViewDir), 3.0);
vec3 atmosphereColor = vec3(0.0, 0.78, 0.83);  // --color-cyan 的 float 值
gl_FragColor = vec4(atmosphereColor * fresnel * 1.4, fresnel * 0.8);
```

**效果規格**：
- 大氣層厚度（視覺）：地球半徑的約 8-12%
- 顏色：從地球邊緣的霓虹青（`#00C8D4`）向外漸變至完全透明
- 發光強度：在飛行弧線抵達地球附近時略微增亮（程序化，用 uniform 控制）

---

### 5.4 飛行弧線的視覺語言

**選型：粒子流 + 細發光線的複合方案**

每條飛行弧線由兩個層次組成：

**Layer 1（主弧線）**：
- Three.js `TubeGeometry` 沿 `QuadraticBezierCurve3`
- 直徑：0.004（地球半徑為 1 的情況下）
- 材質：`MeshBasicMaterial`，顏色 `#00C8D4`，透明度 0.6
- 不使用外發光（反對廉價霓虹感），依靠顏色本身的明度對比

**Layer 2（粒子尾跡）**：
- 沿弧線路徑分布 80-120 個粒子點（`Points`）
- 粒子大小：0.008，隨機抖動 ±0.003
- 顏色：`#20E0EC`（cyan-glow），透明度按距離飛機 sprite 遠近漸變（近亮遠暗）
- 動畫：粒子沿弧線方向以不同速度漂移，製造「數據串流」而非「燈管」的視覺感

**整體效果**：看起來像衛星地面站接收信號的掃描路徑，而非遊戲中的霓虹特效。

### 5.5 飛機圖示的視覺設計

**設計原則**：不是塑膠玩具飛機，是雷達螢幕上的飛行目標回波

**Three.js Sprite 規格**：
- 形狀：極度簡化的飛機俯視輪廓 SVG（轉 Canvas Texture），去除所有細節，保留機翼和機身輪廓
- 尺寸：0.05（地球半徑為 1 的情況下，視覺上約地球直徑的 2.5%）
- 顏色：`#E8EDF2`（--color-foreground），不用純白
- 下方有一個極細的橢圓陰影光暈（`#00C8D4`，blur 8px）

**狀態視覺差異**：
- APPROACHING：飛機後方有 4-6 個漸縮的橢圓同心環（類雷達目標鎖定），按 800ms 週期脈衝
- HOLDING：飛機沿弧線慢速循環，後方有 2 個細小點光粒子跟隨
- ON FINAL：飛機接近地球表面，尺寸視角縮小，前方出現 ILS 進場燈排（見第 7 節）
- TAXIING：飛機在地球表面移動，沿赤道弧線，速度極慢
- ACTIVE：飛機固定在地球表面某節點，周期性脈衝光點（非旋轉）

## 7. 飛機興趣的視覺融入

### 7.1 設計哲學

業主的飛機興趣是「彩蛋層」而非「主題層」。懂的人看到會心一笑，不懂的人以為這就是科技風格的標準元素。以下所有視覺融入都服從這個原則：不影響 B2B 的專業感，但讓真正懂飛機的訪客感到被禮遇。

---

### 7.2 具體視覺元素

**ILS（儀器降落系統）進場燈序列**

當任何一架飛機的狀態切換到 ON FINAL 時，在地球表面對應的城市節點位置，渲染一組微型 ILS 進場燈：
- 7-9 個等距白色點（`#E8EDF2`），沿進場方向排列，向地表延伸
- 最外側 2 個為琥珀色（`#D4940A`），模擬 PAPI 警示燈
- 動畫：燈序從外至內依次閃爍，週期 1.2 秒（真實 Rabbit Light 頻率）

懂航空的人認出這是 ALSF-2 進場燈系統，覺得設計師懂行。不懂的人看到只是「酷炫的落地動畫」。

**雷達環（Radar Sweep）**

地球儀外圍有一個極細的圓環（`stroke-width: 0.5px`，顏色 `#00C8D4`，透明度 0.3），以 4 秒週期旋轉，模擬雷達掃描面。掃描線旋轉時，路過的飛機 sprite 會有短暫（200ms）的亮度提升，暗示「雷達已偵測到該航班」。

**飛行數據卡的航空術語格式**

數據卡使用真實航空無線電通話格式的欄位標籤：

```
FLT-01  AI HUNTER
━━━━━━━━━━━━━━━━━━━━━
STS: APPROACHING     ← Status
ALT: FL021 (v2.1)    ← Altitude = 版本號（FL + 版本號，Flight Level）
PAX: 127             ← Passengers = 活躍用戶
ETA: Q3 2026         ← Estimated Time of Arrival = 下個里程碑
TCKS: LLM · RAG      ← Techs（縮寫 TCKS）
```

`FL021` 這種格式（Flight Level + 版本號）是只有飛機迷才能抓到的隱藏彩蛋。

**左欄指示燈**

每個產品的彩色指示燈不只是純色圓點，而是模擬航空 NOTAM（航行通告）系統的狀態格式：
- ACTIVE / APPROACHING：實心圓點，持續亮
- HOLDING：空心圓點，慢速脈衝（2 秒週期）
- ON FINAL：圓點快速閃爍（0.6 秒週期，模擬急迫降落）

**底部 HUD 的 ATIS 廣播格式**

底部狀態列以 ATIS（自動終端資訊服務）廣播文字格式呈現：

```
[ATIS INFO VICTOR]  SURFACE: OPTIMAL  WIND: 3 ACTIVE FLIGHTS  VIS: 7Y EXP  ALTM: 125 NODES
```

`ATIS INFO VICTOR` 是真實的航空術語（以字母代號標識的 ATIS 廣播識別碼），讓懂的人立刻會心一笑。整個底部 HUD 讀起來像真實的 ATIS 報文。

**導覽列品牌標識**

`COMMAND CENTER` 前方的點標識由純白色圓點替換為雙環同心圓（外環虛線，內環實心），模擬航空雷達站的符號，尺寸約 16x16px。

## 8. 動畫美學

### 8.1 Three.js 3D 動畫整體氛圍

**節奏定位：慢中有精準（Slow with Precision）**

MOTION:9 不意味著快速、閃爍和亢奮。在航管中心的語境中，高動感應該是「系統永遠在動，但永遠是受控的動」。所有動畫都是有機且循環的，沒有突兀的打斷。

類比：不是電玩遊戲的特效，是深夜 CNN 指揮室螢幕牆上的數據流動。

**速度規範**：

| 動畫類型 | 速度 | 感受 |
|---------|-----|-----|
| 地球自轉 | 60 秒 / 圈 | 如衛星圖的地球自轉，觀察者感知到「在動」但不急躁 |
| 飛行弧線粒子流 | 沿弧線 4-8 秒 / 圈（依狀態而異） | 如數據包傳輸 |
| 雷達掃描環 | 4 秒 / 圈 | 真實雷達掃描週期的 2-4 倍慢（增加沉靜感） |
| ILS 進場燈序 | 1.2 秒 / 循環 | 接近真實速度，讓懂的人認出 |
| APPROACHING 脈衝環 | 800ms / 次 | 緊張感但不失控 |
| 飛機 Bob 動畫 | 3 秒上下一個週期 | 模擬飛機在氣流中的輕微顛簸 |
| B2B KPI Count-up | 1.4 秒完成 | 慢一點，讓數字的說服力充分釋放 |

---

### 8.2 光影動態

**地球光影動畫（非動畫，靜止方向光）**

地球的明暗面不隨時間動態變化（避免分散注意力），維持固定的「左上 10 點鐘方向」太陽光位。

**大氣層呼吸（Atmosphere Breathing）**

大氣層 shader 的 `emissiveIntensity` 以 8 秒週期微幅波動（±0.05），製造「地球在呼吸」的生命感。振幅極小，需要專注觀察才能察覺。

**弧線掃光（Arc Sweep Light）**

當飛行弧線的粒子流通過時，弧線底層的 TubeGeometry 的透明度隨粒子位置同步波動，產生「光在線上滑過」的效果，而非靜止的線 + 移動的點。

**UI 面板脈衝（Panel Pulse）**

選中產品的左欄面板邊框：`--color-border-active`（cyan 30% 透明）以 2 秒週期輕微脈衝（透明度 30% 到 50% 再回 30%）。使用 CSS `@keyframes`，不消耗 Three.js 資源。

**HUD 文字動畫（Ticker Update）**

底部 HUD 的 `CONNECTED NODES` 數值每 6-10 秒（隨機）更新一次，更新動作：舊數值向上淡出（translateY -4px + opacity 0），新數值從下方進入（translateY 4px 到 0 + opacity 0 到 1），時長 200ms，製造「系統數據在即時刷新」的感知，而無需實際 API 數據。

---

### 8.3 互動動畫

| 互動 | 動畫 | 時長 | 緩動 |
|-----|-----|-----|-----|
| Hover 飛機 sprite | 飛機尺寸 1.0 到 1.2，脈衝環停止閃爍改為常亮 | 150ms | ease-out |
| 點擊飛機 / 左欄項目 | 地球緩慢旋轉至最佳視角，相機輕微平移 | 800ms | cubic-bezier(0.16, 1, 0.3, 1) |
| Drawer 滑入 | 從右側 translateX(100%) 到 0 | 200ms | ease-out |
| KPI Count-up | 從 0 到目標值 | 1400ms | ease-out（前 70% 快，後 30% 慢） |
| Hover 左欄列表項 | 背景色從 transparent 到 panel-overlay（8% 白），左側邊框加亮 | 100ms | ease-out |
| 視角切換（企業 / 技術） | 右欄內容 fade-out + fade-in，不跳動 | 150ms | ease-in（出）+ ease-out（入） |

### 9.2 視覺風格選型（從 22 種中選出）

依據 anti-generic skill 第七節的設計詞彙庫，本次 V3 採用以下風格組合：

**主要選型：Scroll-Pinned Hero（英雄版型）**

選擇原因：三欄 ATC 佈局本身就是一個「固定的指揮台」概念，整個首頁是一個 pin 住的操作介面而不是向下捲動的敘事頁面。Scroll-pinned 結構讓頁面維持沉浸感，捲動行為觸發右欄內容切換而非頁面滾動，符合 DENSITY:8 的高密度佈局需求。

**次要選型：Bento Grid（右欄）**

選擇原因：右欄的 B2B KPI + 資格認證 + Global Network + Tech Stack 四個區塊在有限空間內需要有機組合，而非等高四行排列。Bento Grid 允許不同高度的磁磚，讓 KPI 數字區塊可以比認證圖示區塊更高，自然傳達資訊優先序。

**輔助選型：Spotlight Border Card（飛行數據卡）**

選擇原因：飛行數據卡在 Three.js 場景中作為 HTML overlay 漂浮在地球前景，需要強烈的視覺存在感又不能喧賓奪主。Spotlight Border Card 在鼠標靠近時邊框發光，讓訪客知道這個卡片是可互動的，同時在平靜狀態下不搶鏡。強調色使用 `--color-cyan-trace`（10% 透明的青色）作為靜止邊框，hover 時提升至 `--color-border-active`（30% 透明）。

**輔助選型：Text Scramble Effect（HUD 文字更新）**

選擇原因：底部 HUD 的數值更新不直接顯示新數字，而是先在 80ms 內對數字字元做 matrix-style 隨機字符亂碼（只用數字字元 0-9，不用英文字母，避免干擾可讀性），再定格為正確數值。這個效果在「DENSITY:8 的儀表板美學」中極為合適，且製造技術感，是「懂行」的細節。

**輔助選型：Animated SVG Line Drawing（ILS 進場燈）**

選擇原因：ILS 進場燈序列從地表延伸的動畫採用 SVG path 的 stroke-dashoffset 動畫自我繪製，在飛機狀態切換到 ON FINAL 時觸發，視覺上像跑道燈被逐一點亮，配合真實的 1.2 秒 Rabbit Light 頻率。這是細緻的「精確美學」而非炫技。
