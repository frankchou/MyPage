## 五個關鍵再設計決策

### 決策 1：補上 B2B 戰績（最高優先）

Gemini 圖最大的遺漏。右欄沒有任何量化 B2B 數據，而這是業主最有說服力的內容（-97% 時間成本、+60% 效率等）。V3 將 B2B KPI 儀表板放在右欄最頂部，首屏可見，Count-up 動畫觸發。

### 決策 2：Hero 文字脫離地球儀

Gemini 圖的 CTA 按鈕疊在地球儀中央，造成視覺競爭。V3 將主標題移至地球儀上方，CTA 移至右欄 KPI 下方的自然閱讀終點，以及右下角固定浮動按鈕。地球儀中央保持乾淨。

### 決策 3：地球儀 Three.js 化（核心升級）

靜態圖片升級為真實 Three.js 3D 地球，帶有：可拖曳旋轉、飛行弧線動畫（QuadraticBezierCurve3）、飛機 Sprite 沿路徑移動、點擊飛機觸發詳情。行動裝置完整降級方案（SVG + CSS 動畫），GPU 能力自動偵測分級。

### 決策 4：雙入口聯動設計

訪客可以從左欄列表點擊產品，也可以直接點擊地球儀上的飛機，兩個入口都觸發相同的聯動效果（左欄高亮 + 右欄切換詳情面板）。降低認知負荷，符合不同訪客的探索習慣。

### 決策 5：完整行動端體驗設計

Gemini 圖完全沒有考慮手機版。V3 設計了完整的 375px 單欄方案：
- 地球儀 -> SVG 全寬卡片
- B2B KPI -> 橫向 scroll snap 卡片
- 產品列表 -> 手風琴 Accordion
- CTA -> 固定底部全寬按鈕（含 iOS Safe Area Inset 適配）

## 與 V1 的差異

V1 是從 PRD 直接出發的概念設計，以 4 頁結構為基礎。

V3 以 Gemini 示意圖為出發點，保留其三欄 ATC 佈局並加以強化：
- V1 未處理地球儀的 3D 互動規格（Three.js 是 V3 新增）
- V1 的首頁缺乏 B2B KPI 首屏位置設計，V3 明確放在右欄最上方
- V1 的行動端設計較籠統，V3 有完整的手風琴 + scroll snap 方案
- V3 新增「雙入口聯動」設計（左欄列表 + 地球飛機 Sprite 同步）
