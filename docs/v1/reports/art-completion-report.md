## 核心設計決策摘要

**決策一：三旋鈕設定為 Variance 8 / Motion 6 / Density 5**
作品集的目標受眾是 CTO/CEO/獵頭，需要強烈個性（Variance 8）與適度動感（Motion 6）傳遞系統感，同時保持高管閱讀舒適度，避免過度密集（Density 5）。

**決策二：Bioluminescent Solarpunk 為最終視覺風格**
從 22 種視覺風格中選定此風格，理由是它處於「純 Cyberpunk」與「極簡主義」之間的差異化地帶，以深暗底色搭配有機自發光元件，既符合航管中心隱喻，又避開競品過飽和霓虹漸層的視覺公式。

**決策三：三色語義分域，嚴格禁止混用**
- 霓虹青 #00f5d4：系統感、在線狀態、CTA（第一印象建立者）
- 琥珀金 #f5a623：B2B 商業穩定、數字權威（企業神經中樞）
- 霓虹綠 #39ff14：AI 生命力、技術前沿（AI 機庫）
三色各司其職，形成清晰的視覺語義地圖，讓使用者在視覺層面感受到網站的雙核心結構。

**決策四：飛機興趣以「航管中心隱喻」融入，而非直接插圖**
雷達掃描圓圈（Hero 右側裝飾元件）、blip 光點、終端機路徑文字、系統狀態列，將飛機迷對精密系統的熱愛轉化為設計語言，懂的人會心一笑，不懂的人只感受到高科技統御感。

**決策五：排除插畫，以動態數字與發光元件作為視覺趣味主體**
插畫會軟化目標受眾所期望的嚴謹感。所有視覺娛樂性由 count-up 動畫、雷達掃描、orb pulse 呼吸效果承擔，符合「展示代替說明」的核心策略。

## 與 PRD 的差異點說明

**差異一：未採納 WebGL 全屏 3D 渲染網格**
PRD 原始規格要求 AI 機庫採用 WebGL 渲染的 3D 數據矩陣。基於效能考量（LCP < 1.5s 的 NFR 要求）與低效能裝置優雅降級原則，美術方向改為 CSS 純代碼點陣網格（linear-gradient 實現），視覺效果相近，但無 WebGL 環境崩潰風險，且降級為靜態 SVG 的條件路徑更穩定。此決策需與工程師及 UX 設計師確認。

**差異二：未為飛機興趣單獨規劃飛機圖像素材**
PRD 中未直接提及飛機圖像融入策略。美術方向決定以「航管中心」作為整體隱喻，而非放置飛機插圖或攝影圖像。此決策基於目標受眾分析：CTO/CEO 更傾向從精密系統語彙中感受信任，飛機圖像反而可能降低作品集的專業重量。

**差異三：og-cover.webp 為新增素材，PRD 未提及**
SEO 與社群分享需要 Open Graph 封面圖，此素材為美術設計師主動補充，不在原始 PRD 規格範圍內，建議列入 V1 實作清單。
