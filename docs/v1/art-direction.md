# Art Direction — Portfolio V1

## 三旋鈕設定（任務開始前確認）

| 旋鈕 | 設定值 | 理由 |
|------|--------|------|
| Design Variance | 8 | 個人作品集需強烈個性，非對稱佈局、雷達視覺元素打破常規 |
| Motion Intensity | 6 | 雷達掃描、count-up 數字、orb 呼吸動畫傳遞系統感，但不過度 |
| Visual Density | 5 | 指標卡與節點卡適度密集，Hero 區保留大量留白維持高管閱讀舒適度 |

---

## 1. 整體美學定位宣言

這個網站不是履歷，是一座全息指揮中樞。

它為一位橫跨政府系統、跨國製造、餐飲集團三大產業的工程師 PM 發聲，同時展示他親手架構並上線 5 個 AI SaaS 產品的真實實力。網站的靈魂是「沉著的統御感」——不是靠炫目特效堆砌，而是靠深邃、剋制的暗色空間，讓琥珀金指標數字與霓虹青技術節點自然發光，讓每一塊數據卡、每一個 AI 節點都散發出彭博終端機與航管中心的冷靜氣度。

目標受眾（CTO / CEO / 獵頭）在進入這個網站的頭三秒，必須感受到：這個人做事有底盤，有數字，有技術，同時敢於前沿。

## 2. 視覺靈感參考方向（3 個）

### 方向 A：深空觀測站（Deep Space Observatory）
- 視覺感受：極暗背景，儀器指針、雷達環、閃爍光點，帶有冷靜的科學精密感
- 色調：近黑深藍底，白綠色冷光儀表，微量琥珀刻度
- 情緒：沉穩、精準、有距離但令人信賴

### 方向 B：彭博/路透金融終端機（Bloomberg Terminal Aesthetic）
- 視覺感受：黑底橙字，等寬字體，格線資料密集排列，無多餘裝飾
- 色調：#0a0a0f 底色，#f5a623 琥珀金數字，灰色輔助文字
- 情緒：高效、權威、數據說話，適合 B2B 企業神經中樞區塊

### 方向 C：生物發光深淵（Bioluminescent Deep Sea）
- 視覺感受：黑暗中自發光的有機體，有生命脈動，帶有 Solarpunk 的生機感
- 色調：深黑底，#39ff14 霓虹綠、#00f5d4 霓虹青作為自發光點
- 情緒：前沿、有生命力、AI 原生感，適合 AI 機庫區塊

## 3. 推薦最終視覺方向

**選型：方向 A（深空觀測站）作為全局主軸，方向 B 強化 B2B 區塊，方向 C 強化 AI 機庫區塊。**

這是一個雙核心設計策略，三方向在同一空間內各司其職，形成「冷靜統御 + 數據說服 + 技術前沿」的完整心理路徑。

### 色彩情緒分析

| 區塊 | 主視覺色 | 情緒目標 |
|------|---------|---------|
| 全域底色 | 深曜黑 #0a0a0f | 航管中心暗室沉浸感 |
| Hero 雷達 / 導覽列 | 霓虹青 #00f5d4 | 系統在線、科技前沿、第一印象鮮明 |
| B2B 數據卡 | 琥珀金 #f5a623 | 商業利潤、穩定可信、高管熟悉的財務色 |
| AI 機庫節點 | 霓虹綠 #39ff14 | 生命力、AI 原生、技術節點的自發光感 |
| 靜態文字 | 近白 #e8e8f0 | 高對比閱讀舒適，保留暖調避免冷硬 |
| 次要文字 | 藍灰 #8888a8 | 資訊層級明確，不搶奪主視覺注意力 |

### 光影處理方式

- 光源邏輯：光從內部元件向外散，不是從上方打光。每個發光元件（數字、節點 orb、導覽列 logo）都帶有 `box-shadow` 或 `text-shadow` 的 glow 擴散。
- 光暈（Glow Halo）：Hero 區域右側霓虹青光暈、左下琥珀金光暈，模擬「儀表板後方的環境光洩漏」。
- 毛玻璃（Glassmorphism）層：所有資訊卡片採半透明毛玻璃，`rgba(255,255,255,0.05)` 底色 + `backdrop-filter: blur(12px)`，1px 邊框帶有微光折射，傳遞「懸浮在空間中的 HUD 面板」感。
- 漸層方向：頁面整體由上到下，底色從 #0a0a0f 微微偏移至 #0d0d14，避免純平無深度。

### 材質感

| 材質 | 用於 | 具體實現 |
|------|------|---------|
| 毛玻璃（Glassmorphism） | 所有資訊卡、導覽列、側面板 | backdrop-filter blur + 半透明背景 + 1px 微光邊框 |
| 金屬折射 | 琥珀金強調數字、B2B 卡頂部高光線 | 2px 頂部漸層線（transparent to amber to transparent） |
| 發光有機體 | AI 機庫節點 orb | 雙層 pulse 圓圈動畫 + glow box-shadow |
| 點陣網格 | Hero 背景、機庫背景 | CSS linear-gradient 1px 格線，極低不透明度 |

### 空間感

- 近景：發光元件（導覽 logo 點、orb、數字）
- 中景：毛玻璃卡片、雷達圈
- 遠景：網格背景、光暈散射
- 深度感：雷達元件置於 Hero 右側，設定 18% 低透明度，強化「透過玻璃看向遠處儀表」的縱深。

## 4. 飛機興趣的視覺融入

業主的「看飛機」興趣不應以字面圖像直接呈現（如飛機插圖），而應以「航管中心」作為整個網站的視覺隱喻，將飛機文化的精髓轉化為設計語彙。

### 如何融入而不突兀

飛機迷欣賞的不是飛機本身，而是飛機系統背後的精密、邏輯與掌控感——航管員的雷達螢幕、儀表板的數字、起飛倒數的節奏感。這些元素正好與「展示技術統御力」的作品集目標高度吻合。

### 具體視覺元素建議

| 視覺元素 | 設計表現 | 位置 |
|---------|---------|------|
| 雷達掃描圓圈 | 四層同心圓 + 十字準星 + 旋轉扇形掃描光 + 隨機光點（blip） | Hero 右側，低透明度裝飾 |
| 航管中心暗室感 | 全域深曜黑底色，僅強調元件發光 | 全頁 |
| 系統狀態列 | `SYSTEM_ONLINE` / `5_AI_SAAS_LIVE` 等類終端機文字 | Hero 頂部 status bar |
| 麵包屑路徑 | `Root > AI_Skunkworks > Multi-Agent_Architecture` 終端機風格 | 側面板頂部 |
| 節點光點（blip） | 雷達上四個脈衝光點，模擬追蹤中的飛行目標 | Hero 雷達元件 |
| B2B 指標 | 「-97%」等大型數字，對應儀表板讀值 | 企業神經中樞區塊 |

> 飛機興趣融入的核心原則：讓懂的人一眼看出意境，但不需要說明也能感受到精密與掌控感。

## 5. 差異化美學策略

### 與常見黑色科技作品集的差異化切入點

大多數深色科技作品集會犯以下三個毛病：

1. **AI 紫藍霓虹漸層**：紫色按鈕輝光、彩虹漸變背景，已是過度飽和的視覺公式。
2. **動態過度**：粒子系統、3D 旋轉地球、WebGL 全屏渲染，技術展示喧賓奪主，內容反而被忽略。
3. **缺乏功能性語言**：華麗但空洞，看不出業主真正做過什麼。

### 本作品集的三個差異化切入點

**第一：雙色系統，非單一霓虹**
使用琥珀金（B2B 商業穩定）+ 霓虹青（系統在線）+ 霓虹綠（AI 生命力）三色分域管理，每個顏色有明確的語義角色，避免混用導致視覺噪音。競品往往將所有強調元素用同一個紫色或藍色，失去層次。

**第二：數字本身就是視覺**
用彭博終端機風格的等寬字體巨型數字（-97%、100%、2 國 5 部門）作為視覺主角，而不是插圖或圖表。數字本身帶有強烈的視覺重量與可信度，讓作品集的「展示」勝過「說明」。

**第三：剋制的發光，而非滿屏輝光**
光源只來自功能性元件本身，背景保持純淨的深曜黑。這與大多數競品「全頁撒光」的做法相反，反而讓每一個發光點更具分量，符合高管對「信號雜訊比」的本能判斷。

## 7. 圖示 / 插畫風格建議

### 視覺風格選型

**從 22 種視覺風格中選型：Bioluminescent Solarpunk（生物發光 Solarpunk）**

選擇理由：

1. **符合雙核心語意**：Solarpunk 代表「高效且有生命力的未來技術」，完美承載 B2C AI SaaS 產品的有機、前沿感；深色 Bioluminescent（生物發光）基底則與全局深曜黑底色無縫整合。
2. **區隔競品**：純 Cyberpunk（過度飽和霓虹）與純極簡主義（缺乏個性）是最常見的兩極，Bioluminescent Solarpunk 恰好在兩者之間，既有科技感又有生命感。
3. **避開設計系統禁區**：此風格不使用「AI 紫藍霓虹漸層」，不依賴彩虹漸變，符合色彩禁止模式規範。
4. **飛機興趣的視覺呼應**：航管雷達 + Bioluminescent 自發光元件，在暗室中尋找發光訊號的視覺語言，與業主觀看飛機時的心境高度吻合。

### 圖示風格

採用**純 SVG Inline 線條圖示**，不使用外部圖示庫（降低依賴、提升控制力）。

| 屬性 | 規格 |
|------|------|
| 線條風格 | 等寬線條（stroke-width: 2），圓角線頭（stroke-linecap: round） |
| 尺寸 | 12px / 14px / 16px（依場景），容器 24x24px viewport |
| 色彩 | 繼承父元素 `currentColor`，隨情境自動對應 cyan / amber / green |
| 格式 | SVG inline，不使用外部 .svg 檔引入（減少 HTTP 請求） |

### 素材規格清單

| 素材名稱 | 用途 | 尺寸 | 格式 | 備註 |
|---------|------|------|------|------|
| `icon-calendar.svg` | CTA 預約按鈕圖示 | 14x14px（viewport 24x24） | SVG inline | stroke: currentColor (#00f5d4) |
| `icon-close.svg` | 節點側面板關閉按鈕 | 14x14px（viewport 24x24） | SVG inline | stroke: currentColor |
| `icon-star.svg` | 架構流程步驟圖示 | 12x12px（viewport 24x24） | SVG inline | stroke: #39ff14 |
| `icon-download.svg` | PDF 下載次要連結 | 12x12px | SVG inline | muted color |
| `radar-asset.svg` | Hero 右側雷達裝飾元件 | 480x480px（桌面）/ 300x300px（平板） | SVG inline + CSS animation | 四環 + 十字 + 掃描扇形 + blip 光點 |
| `grid-bg-hero.svg` | Hero 點陣網格背景 | 48x48px tile（CSS repeat） | CSS 純代碼實現（linear-gradient） | 無需獨立 SVG 檔 |
| `grid-bg-skunkworks.svg` | AI 機庫點陣網格背景 | 64x64px tile | CSS 純代碼實現 | 霓虹綠色，低透明度 |
| `topology-lines.svg` | B2B 卡片 hover 背景拓撲連線 | 卡片等寬等高 | CSS 純代碼實現（absolute div） | 琥珀金，8% 透明度 |
| `orb-node.svg` | AI 機庫節點 orb（發光圓點） | 40x40px + 12px 外圈 | CSS 純代碼 + animation | 雙層 pulse 圓圈，霓虹綠 |
| `og-cover.webp` | Open Graph 社群分享封面圖 | 1200x630px | WebP | 黑底，F.CHOU.SYS logo，副標題，霓虹青色調 |
| `favicon.svg` | 瀏覽器 tab 圖示 | 32x32px | SVG | 霓虹青點 + F 字母，極簡 |

### 插畫方向說明

本作品集設計上**刻意不使用插畫**（illustration）作為視覺元素，原因如下：

1. 插畫會軟化整體的科技統御感，與「指揮中樞」的定位不符。
2. 目標受眾（CTO/CEO/獵頭）更信任數字與結構圖，而非插畫帶來的消費感。
3. 所有「視覺趣味性」由動態元件（雷達掃描、count-up 數字、orb pulse）提供，不需插畫補充。

若未來 V2.0 需要插畫，建議方向：**極細線條的技術藍圖風格（Blueprint Style）**，使用霓虹青 #00f5d4 在深底色上，模擬工程師手繪 schematic 的感覺。
