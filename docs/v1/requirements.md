## 1. 專案目標摘要

本專案旨在打造一座個人作品集網站，以「全領域航管指揮中心」為視覺隱喻，透過雙核心架構同時展示業主的 B2B 企業系統管理戰績與 5 個自製 AI SaaS 產品的底層技術能力。

目標是取代傳統 PDF 履歷，讓造訪的決策者（CTO、CEO、高階獵頭）在 3 分鐘內形成「此人兼具落地百億企業系統的 PM 穩定性，與親手打造 AI 產品的技術前瞻性」的精確認知，進而主動預約諮詢對談或發出合作邀約。

核心價值主張：**用網站本身的精緻度與技術深度，作為業主能力最真實、最無法偽造的展示載體。**

---

## 2. 目標使用者（Personas）

### Primary Persona：高階決策者
- **角色**：CTO、CEO、高階技術獵頭
- **情境**：在評估數位轉型顧問、技術 PM 或 AI 架構師人選時，主動或被推薦造訪本站
- **核心問題**：「這個人能否主導我們公司的 AI 轉型？他有沒有真正落地過複雜系統？」
- **行為模式**：停留時間短（3-5 分鐘）、優先掃描量化成效與技術架構深度
- **成功條件**：看完首頁雙核心區塊後，主動點擊預約諮詢按鈕

### Secondary Persona：中階篩選者
- **角色**：技術主管、HR Business Partner、招募負責人
- **情境**：被上級指派進行初步篩選，或在 LinkedIn 搜尋時發現本站
- **核心問題**：「他的技術棧符合我們的需求嗎？這份資料可以直接轉給高層嗎？」
- **成功條件**：成功找到技術細節後，下載 PDF 或複製網址轉發給決策者

## 4. 驗收標準（Acceptance Criteria）

**M-03 AI 兵工廠節點展示**

- Given 使用者點擊任一 AI 技術節點（例如 Node 01 AI Hunter）
- When 點擊事件觸發
- Then 0.5 秒內從畫面側邊滑出半透明說明板，展示底層架構描述與技術棧；頁面不跳轉不重新載入

**M-04 預約諮詢 CTA（降級情境）**

- Given Calendly API 無法連線（逾時超過 5 秒）
- When 使用者點擊 CTA 按鈕
- Then 表單區域顯示備用文案與可點擊的 Email 連結

**M-10 效能基準**

- Given 在 Chrome 模擬 4G 網路，以 Lighthouse 測試首頁
- When 執行效能測試
- Then LCP < 1.5 秒；Lighthouse Performance Score >= 75

## 5. 非功能性需求（NFRs）

- 效能：LCP < 1.5s，低效能裝置降級至靜態 SVG
- RWD：Desktop 1280px+ / Mobile 375px+
- 無障礙：WCAG AA（深色模式對比度）
- SEO/AIO：語意化 HTML5、Schema.org、TechArticle 結構化資料

---

## 6. 本版範圍（V1 Scope）決策

V1 聚焦首頁完整雙核心體驗，含 Hero、B2B KPI、AI 節點展開。企業深度頁與 Terminal 頁列為 Should Have，視工程時程決定是否納入。
